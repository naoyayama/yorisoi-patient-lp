# LP用イラスト 必要項目一覧

現在のLP構成に合わせて、差し込み可能なイラストの項目をまとめました。デザイン依頼や素材選定の際のチェックリストとしてご利用ください。

---

## 推奨仕様（共通）

| 項目 | 推奨 |
|------|------|
| 形式 | PNG（透過可）または WebP |
| アスペクト比 | 使用箇所に応じて 1:1 または 4:3 |
| 解像度 | 幅 400〜800px 程度（Retina 考慮なら 800px） |
| 配置先 | `public/images/` に配置後、コード側でパスを指定 |

---

## 1. ヒーロー（ファーストビュー）

| 項目 | 内容 |
|------|------|
| **推奨ファイル名** | `hero.png` または `hero.jpg` |
| **使用箇所** | Hero セクション（キャッチの横 or 下のメインビジュアル） |
| **イラストの内容** | ・「診察・医療」と「身近なツール（スマホ）」がつながり、**安心**が届くイメージ。<br>・例：スマホと手（タッチ・支え）、スマホから通知やメモが届くイメージ、患者・家族が安心している様子など。 |
| **トーン** | 安心感・清潔感。堅くならず、親しみやすい。 |
| **備考** | 全面背景にするか、テキストの横に1枚置くかで構図を調整。 |

---

## 2. 悩み（もやもや）セクション — 3点

各カード用の小さいイラスト（アイコン程度のサイズで使用想定）。

### 2-1. 患者さんの悩み

| 項目 | 内容 |
|------|------|
| **推奨ファイル名** | `problem-patient.png` |
| **使用箇所** | 「患者さん」カード内（ラベル横 or カード上部） |
| **イラストの内容** | 「説明が難しくて覚えきれない」「帰ってからあれ、何だっけ？」という**もやもや・不安**。<br>・例：頭の上に「？」や雲、眉をひそめた顔、メモと頭を抱える人など。 |
| **トーン** | 共感・あるある。暗くしすぎない。 |

### 2-2. パパ・ママの悩み

| 項目 | 内容 |
|------|------|
| **推奨ファイル名** | `problem-parent.png` |
| **使用箇所** | 「パパ・ママ」カード内 |
| **イラストの内容** | 「子どもが泣いていて聞き逃したかも」「ケアで手一杯」という**育児と通院の両立**。<br>・例：赤ちゃん・幼児を抱えた親、もう片方でメモやスマホを持とうとしている様子など。 |
| **トーン** | 大変さは伝えつつ、温かみのある親子の雰囲気。 |

### 2-3. ご家族の悩み

| 項目 | 内容 |
|------|------|
| **推奨ファイル名** | `problem-family.png` |
| **使用箇所** | 「ご家族」カード内 |
| **イラストの内容** | 「離れて暮らす親の通院内容を共有したい」という**家族の気づかい・つながり**。<br>・例：複数人（親・子・祖父母）の家族、離れた場所同士をスマホ・メッセージでつなぐ構図など。 |
| **トーン** | 心配だが、支え合いが感じられるもの。 |

## 3. 解決策（AI要約）セクション

| 項目 | 内容 |
|------|------|
| **推奨ファイル名** | `solution-summary.png` |
| **使用箇所** | 「診察内容を、AIがやさしく要約」セクション（見出し下 or 3つのカードの横に1枚） |
| **イラストの内容** | **AIが診察内容を要約する**イメージ。<br>・例：書類・メモがやさしい言葉のリストに変わる、スマホに要約が表示される、ロボット/AIがメモを手渡すなど。 |
| **トーン** | 正確さとやさしさ。「難しい説明が、自分用の言葉でまとまる」感。 |

## 4. ベネフィット（安心に）セクション — 2点

### 4-1. 患者さん向け

| 項目 | 内容 |
|------|------|
| **推奨ファイル名** | `benefit-patient.png` |
| **使用箇所** | 「患者さんへ」ブロック（テキストの横の画像エリア） |
| **イラストの内容** | **メモを取らなくていい安心感**。先生の目を見て話せる、リラックスした患者像。<br>・例：穏やかな表情の人物、手は空っぽ or スマホをポケットに、メモ帳を持たない姿など。 |
| **トーン** | 安心・リラックス。肩の力が抜けた感じ。 |

### 4-2. パパ・ママ向け

| 項目 | 内容 |
|------|------|
| **推奨ファイル名** | `benefit-parent.png` |
| **使用箇所** | 「パパ・ママへ」ブロック（テキストの横の画像エリア） |
| **イラストの内容** | **子どもが泣いていても大丈夫**。AIが代わりに聞いているので、親は子どもに集中できる。<br>・例：子どもをあやしている親、スマホは台の上などで「聞いている」存在として。 |
| **トーン** | 共感と安心。「大丈夫」が伝わる明るさ。 |

## 5. 使い方（3ステップ）セクション — 3点

各ステップの番号の代わり、または番号と並べて表示するイラスト。

### 5-1. STEP 1 開始

| 項目 | 内容 |
|------|------|
| **推奨ファイル名** | `step-start.png` |
| **使用箇所** | STEP 1「開始」のビジュアル |
| **イラストの内容** | **録音を始める**瞬間。スマホを置くだけ・タップするだけの手軽さ。<br>・例：手がスマホの「録音ボタン」に触れる、スマホから音波が広がるなど。 |
| **トーン** | シンプルで分かりやすい。「難しくない」が伝わる。 |

### 5-2. STEP 2 診察

| 項目 | 内容 |
|------|------|
| **推奨ファイル名** | `step-consultation.png` |
| **使用箇所** | STEP 2「診察」のビジュアル |
| **イラストの内容** | **診察シーン**。いつも通り先生とお話ししている、AIが聞いているイメージ。<br>・例：医師と患者が穏やかに会話、スマホは脇に置いてあるなど。 |
| **トーン** | 穏やか・安心。威圧感のない診察室。 |

### 5-3. STEP 3 完了

| 項目 | 内容 |
|------|------|
| **推奨ファイル名** | `step-done.png` |
| **使用箇所** | STEP 3「完了」のビジュアル |
| **イラストの内容** | **完了・届いた**という達成感。<br>・例：スマホにLINEの通知・要約が届く、大きなチェックマーク、完了の丸印など。 |
| **トーン** | すっきり・安心。「うまくいった」がひと目で分かる。 |

## 6. 信頼（厳重に守ります）セクション — 3点

### 6-1. 匿名化処理

| 項目 | 内容 |
|------|------|
| **推奨ファイル名** | `trust-security.png` |
| **使用箇所** | 「匿名化処理」カードのアイコン部分 |
| **イラストの内容** | **守る・保護・セキュリティ**。<br>・例：盾、鍵、守られたデータ・人物のイメージ。 |
| **トーン** | 信頼感・堅実さ。怖くならない程度に「しっかり守る」。 |

### 6-2. 目的外利用なし

| 項目 | 内容 |
|------|------|
| **推奨ファイル名** | `trust-privacy.png` |
| **使用箇所** | 「目的外利用なし」カードのアイコン部分 |
| **イラストの内容** | **目的外に使わない・第三者に渡さない**。<br>・例：鍵、ロック、×印、データが枠の中に閉じられているイメージなど。 |
| **トーン** | 明確で分かりやすい。「利用範囲が限られている」安心感。 |

### 6-3. 医療機関との連携

| 項目 | 内容 |
|------|------|
| **推奨ファイル名** | `trust-doctor.png` |
| **使用箇所** | 「医療機関との連携で、さらに安心」の緑ブロック内（テキストの横 or 上部） |
| **イラストの内容** | **信頼できる医師・医療者**。AI要約を確認するプロの存在。<br>・例：白衣の医師、穏やかに頷く、カルテやタブレットを確認する姿など。 |
| **トーン** | 信頼感とやさしさ。威圧感のない「味方」の医師像。 |

## 優先度（予算・工数に応じて）

| 優先度 | 項目 | 理由 |
|--------|------|------|
| 高 | hero.png | ファーストビューの印象を決める |
| 高 | benefit-patient.png, benefit-parent.png | ペルソナ別の安心感を視覚化 |
| 高 | step-start.png, step-consultation.png, step-done.png | 利用の流れが直感的に伝わる |
| 中 | problem-patient.png, problem-parent.png, problem-family.png | 共感を強化 |
| 中 | solution-summary.png | サービス価値の核心を補足 |
| 低 | trust-security.png, trust-privacy.png, trust-doctor.png | 現状はSVGアイコンでも代用可能 |

画像を用意いただいたら、`public/images/` に配置し、各コンポーネントで `<img src="/images/〇〇.png" />` を差し込む形で反映できます。必要であれば、画像パスを組み込んだコード例も用意できます。
