# Hero セクション — 画像ベース設計アドバイス

現状の「テキスト中央＋小さなイラスト枠」から、**画像を主役にした Hero** に変えるための方針と実装のポイントです。

---

## 1. レイアウトパターン（選ぶと進めやすい）

画像を主役にする代表的な形です。**どれを目指すか**を決めると、撮影・デザイン・実装が揃います。

| パターン | 説明 | 画像の役割 | 向いている訴求 |
|----------|------|------------|----------------|
| **A. 全面ヒーロー画像** | ファーストビュー全体を1枚の画像（背景）にし、その上にキャッチコピーとCTAを重ねる | 背景として全画面 | 感情・世界観を一瞬で伝えたい |
| **B. 左右スプリット** | 左（または右）に画像、反対側にテキスト＋CTA。PCで横並び、SPで縦積み | メインのビジュアルとして半分を占有 | サービス像・人物・画面を見せたい |
| **C. 画像下段** | 上にキャッチ＋CTA、下に大きく1枚の画像（スマホ画面・サービスイメージ等） | テキストの下で「見せる」ブロック | アプリ画面・Before/After を見せたい |
| **D. 背景＋オーバーレイ** | 写真やイラストを背景にし、暗い/白いオーバーレイでテキストを読みやすく | 雰囲気づくり＋可読性の両立 | 病院・患者・家族の「空気」を出したい |

**よりそい**のように「安心・身近・医療」を伝えるなら、**B（左右スプリット）** か **D（背景＋オーバーレイ）** が扱いやすく、画像ベースにしやすいです。

---

## 2. 画像の内容・トーン

- **写すものの例**: スマホでLINEを見る患者・家族、診察室の雰囲気（個人が特定できない程度）、待合室でリラックスする人、スマホとメモが並んだ「記録される安心」のイメージなど。
- **避けたいこと**: 個人が特定できる顔、過度に暗い・不安を煽る構図。医療機器ばかりで冷たい印象にならないように。
- **権利**: 自作・素材サイト（商用可）・フォトグラファー依頼など、利用許諾をはっきりさせておく。

---

## 3. 技術仕様の目安

| 項目 | 推奨 |
|------|------|
| **形式** | WebP（推奨）または JPEG / PNG |
| **アスペクト比** | **16:9** または **3:2**（横長）。全面背景なら **2:1** も可。 |
| **解像度** | 幅 **1920px** を目安（Retina 対応）。SP用に **1080px** 幅の別画像を用意すると軽量。 |
| **ファイル配置** | `public/images/hero/` などに整理すると管理しやすい。 |

**レスポンシブ**  
- 同じ画像を幅100%で使う場合は、**重要部分が中央**に来るようにトリミングしておく。  
- PC用とSP用で**構図が違う**場合は、`<picture>` で `hero-desktop.jpg` と `hero-mobile.jpg` を出し分けると安全。

---

## 4. 実装のポイント

### 画像の置き方

- **背景として使う場合**: `background-image` でセクション全体に指定。  
  - テキストの可読性のため、`linear-gradient` でオーバーレイ（例: 白や黒の半透明）を重ねる。
- **要素として使う場合**: `<img>` で配置。  
  - `object-fit: cover` でエリアを埋める、または `object-contain` で全体を見せる。

### アクセシビリティ

- `alt` は必ず設定。「よりそいの利用イメージ」「診察後の安心シーン」など、内容が分かる短い文言に。
- 背景画像で意味がある場合は、同じ内容を `aria-label` や見えないキャプションで補足してもよい。

### 表示速度

- ヒーロー画像は LCP に関わるので、`loading="eager"`（デフォルト）のままでOK。  
- 可能なら **WebP** と **適切な幅（例: 1920px）** で配信し、`<picture>` でフォールバックを用意。

---

## 5. 次のステップ

1. **レイアウトを決める**: A〜D のどれに近づけるか（または組み合わせ）を決める。  
2. **画像を用意する**: 上記のトーン・仕様で1枚（または PC/SP で2枚）準備。  
3. **コンポーネントを差し替える**: `Hero.tsx` を、選んだパターンに合わせて画像ベースの構造に変更する。

必要であれば、**B（左右スプリット）** または **D（背景＋オーバーレイ）** を想定した `Hero.tsx` の具体的なコード例も用意できます。希望のパターンがあれば教えてください。
